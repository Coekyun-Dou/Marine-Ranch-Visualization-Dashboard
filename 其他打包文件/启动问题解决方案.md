## 🐛 闪退的常见原因

### 原因 1: 不在正确的目录 ⭐最常见
**现象**: 双击后一闪而过

**检查方法**:
1. 右键点击 bat 文件所在的文件夹
2. 查看是否有以下文件夹：
   - ✅ backend
   - ✅ frontend
   - ✅ package.json

**解决**:
- 确保 bat 文件在项目根目录（与 backend、frontend 同级）

### 原因 2: Node.js 未安装
**现象**: 提示"不是内部或外部命令"

**检查方法**:
1. 按 `Win + R`
2. 输入 `cmd` 回车
3. 输入 `node -v` 回车
4. 如果显示版本号 = 已安装 ✅
5. 如果提示"不是命令" = 未安装 ❌

**解决**:
```
1. 访问 https://nodejs.org/
2. 下载 LTS 版本（18.x 或 20.x）
3. 安装时勾选 "Add to PATH"
4. 安装完成后 **必须重启电脑**
5. 重新运行诊断工具验证
```

### 原因 3: 依赖未安装
**现象**: 提示找不到模块

**检查方法**:
- 查看是否存在 `backend\node_modules` 文件夹
- 查看是否存在 `frontend\node_modules` 文件夹

**解决**:
```bash
# 方式1：使用脚本（会自动安装）
双击 启动.bat

# 方式2：手动安装
# 打开命令提示符
cd backend
npm install

cd ..
cd frontend
npm install
```

### 原因 4: 缺少 axios（AI 功能需要）
**现象**: 后端启动失败，提示找不到 axios

**解决**:
```bash
双击运行：安装axios.bat
```

### 原因 5: 端口被占用
**现象**: 提示端口 3000 或 5173 已被使用

**解决**:
```bash
双击运行：关闭占用端口.bat
```

或手动清理：
```cmd
# 打开命令提示符（管理员）
netstat -ano | findstr :3000
netstat -ano | findstr :5173
# 记住 PID（最后一列的数字）

taskkill /F /PID <进程ID>
```

## 🆘 常见错误信息及解决

### 错误 1: `'node' 不是内部或外部命令`
**原因**: Node.js 未安装或未添加到环境变量

**解决**:
1. 卸载旧版 Node.js（如果有）
2. 重新下载安装 https://nodejs.org/
3. 安装时勾选 "Add to PATH"
4. **重启电脑**
5. 打开 cmd 输入 `node -v` 验证

### 错误 2: `Cannot find module 'express'`
**原因**: 后端依赖未安装

**解决**:
```cmd
cd backend
npm install
```

### 错误 3: `Cannot find module 'axios'`
**原因**: axios 未安装（AI 功能需要）

**解决**:
```cmd
双击运行：安装axios.bat
```

### 错误 4: `Error: listen EADDRINUSE: address already in use :::3000`
**原因**: 端口被占用

**解决**:
```cmd
双击运行：关闭占用端口.bat
```

### 错误 5: `npm ERR! code ENOENT`
**原因**: 不在正确的目录

**解决**:
- 确保在项目根目录运行
- 检查是否有 backend 和 frontend 文件夹
