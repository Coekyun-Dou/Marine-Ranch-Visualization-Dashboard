# 🐛 AI 助手页面无法访问 - 解决方案

## 🎯 问题诊断

如果你无法访问 AI 助手页面（http://localhost:5173/ai-assistant），可能的原因：

### 原因 1: axios 依赖未安装 ⭐最可能
AI 功能需要 axios 库来调用 DeepSeek API。

### 原因 2: 服务未完全启动
后端或前端服务可能还在启动中。

### 原因 3: 浏览器缓存问题
旧版本的页面可能被缓存。

### 原因 4: 路由配置问题
前端路由可能配置不正确。

## 🔍 详细排查步骤

### 检查 1: axios 是否安装

**方法 A - 查看文件**:
检查是否存在：`backend\node_modules\axios` 文件夹

**方法 B - 使用命令**:
```cmd
cd backend
npm list axios
```

**结果判断**:
- ✅ 如果显示版本号 → axios 已安装
- ❌ 如果显示 "empty" 或错误 → 需要安装

**修复**:
```cmd
cd backend
npm install axios
```

---

### 检查 2: 后端是否正常启动

**查看后端终端窗口**，应该显示：
```
🌊 智慧海洋牧场可视化平台后端服务
📡 HTTP服务: http://localhost:3000
客户端连接成功: xxxxx
```

**如果没有这些信息**:
- 可能启动失败
- 查看红色错误信息
- 常见错误：`Cannot find module 'axios'` → 安装 axios

---

### 检查 3: 前端路由是否配置

**检查文件** `frontend\src\router\index.js`，应该包含：
```javascript
{
  path: '/ai-assistant',
  name: 'AIAssistant',
  component: () => import('../views/AIAssistant.vue'),
  meta: { title: 'AI管理助手', requiresAuth: true }
}
```

**如果没有**：路由配置可能被修改了，需要重新添加。

---

### 检查 4: 浏览器控制台错误

1. 访问 http://localhost:5173/ai-assistant
2. 按 `F12` 打开开发者工具
3. 查看 Console（控制台）标签

**常见错误**:

#### 错误 A: `Failed to fetch component`
**原因**: AIAssistant.vue 文件可能有语法错误  
**解决**: 检查文件是否完整

#### 错误 B: `Network Error`
**原因**: 后端未启动或端口不对  
**解决**: 确保后端在 3000 端口运行

#### 错误 C: `Module not found: axios`
**原因**: 后端缺少 axios  
**解决**: `cd backend && npm install axios`
